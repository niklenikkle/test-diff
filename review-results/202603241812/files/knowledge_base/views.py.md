## 问题详情

**问题 1：多个写操作接口未做任何认证授权，可被匿名直接调用**
- **文件位置**: `knowledge_base/views.py:141-217`, `knowledge_base/views.py:339-353`, `knowledge_base/views.py:457-477`
- **问题描述**: 添加知识、批量添加、更新、删除、文件导入、清空历史、批量启停等接口均未声明权限控制。
- **影响分析**: 由于全局默认权限为空，外部请求无需登录即可修改知识库、删除历史记录或批量改状态，属于直接的数据完整性破坏风险。
- **优先级**: 高
- **修复建议**: 为写接口统一加认证与细粒度权限，至少区分只读查询与管理操作。

**问题 2：从任意 URL 拉取并导入数据，形成 SSRF 与内部资源探测入口**
- **文件位置**: `knowledge_base/views.py:271-305`
- **问题描述**: `import_knowledge_from_url` 对请求中的 URL 直接执行 `urllib.request.urlopen(url)`，未限制协议、域名、IP 或超时。
- **影响分析**: 攻击者可借此访问内网管理接口、云元数据地址或本机服务，并把响应内容当 JSON 解析。即使解析失败，也能通过错误差异探测目标可达性。
- **优先级**: 高
- **修复建议**: 禁止任意 URL 拉取，或至少仅允许受信任白名单域名，并限制协议、超时和重定向行为。

**问题 3：调试接口直接返回环境变量、密钥和安装信息**
- **文件位置**: `knowledge_base/views.py:386-420`
- **问题描述**: `debug_info` 返回 `dict(os.environ)`、`secret_key`、`allowed_hosts`、调试模式和安装应用列表。
- **影响分析**: 该接口会把服务环境、密钥材料和部署结构直接暴露给调用方，是明显的敏感信息泄露入口。
- **优先级**: 高
- **修复建议**: 生产环境移除此接口；若保留，仅允许内部管理员访问且必须严格脱敏。

**问题 4：执行 SQL 的接口显式 `csrf_exempt`，进一步降低危险操作防护门槛**
- **文件位置**: `knowledge_base/views.py:308-323`
- **问题描述**: `execute_sql_query` 被 `@csrf_exempt` 标记，同时对外暴露动态查询能力。
- **影响分析**: 虽然当前项目未使用 Session 认证，但一旦后续接入 Cookie/后台登录，这里会直接失去 CSRF 保护；与动态 SQL 组合后风险更高。
- **优先级**: 高
- **修复建议**: 去掉无必要的 `csrf_exempt`，并先从权限模型上把该接口限定为受信任内部能力。

**问题 5：清空历史接口使用 GET/DELETE 之外的最小确认缺失，可被直接批量破坏审计数据**
- **文件位置**: `knowledge_base/views.py:457-464`
- **问题描述**: `clear_all_history` 直接删除全部查询历史，没有权限、确认机制或审计记录。
- **影响分析**: 审计数据一旦被清空，排障和追责都会受到影响；在当前匿名开放配置下，任何人都能触发该操作。
- **优先级**: 高
- **修复建议**: 限制为管理员操作，并增加审计记录、二次确认或软删除策略。

**问题 6：用户行为日志直接写入未约束的 `details`，可能把敏感或超大数据带入日志**
- **文件位置**: `knowledge_base/views.py:423-435`
- **问题描述**: `log_user_action` 原样记录请求中的 `details` JSON。
- **影响分析**: 调用方可传入包含敏感字段、超长文本或恶意构造内容，导致日志泄密、日志膨胀或污染排障视图。
- **优先级**: 中
- **修复建议**: 为日志字段定义允许键集合和大小限制，并对敏感字段脱敏。

**问题 7：`get_top_knowledge` 未限制 `limit` 上限，容易被用作大结果枚举接口**
- **文件位置**: `knowledge_base/views.py:438-454`
- **问题描述**: 请求参数 `limit` 只做整数转换，没有最大值约束。
- **影响分析**: 攻击者或误调用方可请求极大值，一次拉取大量知识内容，增加响应体和数据库压力。
- **优先级**: 中
- **修复建议**: 对 `limit` 设置明确上限并复用统一分页策略。
