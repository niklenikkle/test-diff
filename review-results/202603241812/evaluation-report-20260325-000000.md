## 优化建议

1. **加强“日志安全”规则，不只看敏感信息泄露，还要显式检查日志注入 / log forging。**
   - 尤其是 `f-string + user input + logger.*()`、允许换行、结构化日志字段未约束等模式。

2. **把高风险配置项单独强制落项。**
   - 至少包含：`DEBUG=True`、`SECRET_KEY` 硬编码、`ALLOWED_HOSTS=['*']`、`CORS_ALLOW_ALL_ORIGINS=True`、默认无认证权限。
   - 避免“只在建议里提到，但没进正式问题表”。

3. **对重复漏洞模式增加“多位置展开”能力。**
   - 同类弱 SQL 校验、弱路径校验、弱认证边界如果在多个文件重复出现，应按“一个模式 + 所有命中位置”输出，而不是只报其中一个位置。

4. **将“新增有效发现”和“基准命中项”分栏展示。**
   - 这样既保留扩展发现能力，又不会影响基准评估时的清晰度。

5. **降低少量非核心提醒的权重。**
   - 像 `claude_automation.py` 的路径硬编码，在漏洞评估场景里应放入“低优背景观察”，不要和核心安全问题混排。
