# Code Reviewer Evals Grading Guide

本文件用于说明如何评估 `code-reviewer` 在 `evals/evals.json` 中各测试用例上的表现。

## 评估目标

评估重点不是“输出看起来是否很多”，而是判断该 skill 是否：

1. 命中了值得现在处理的高价值问题
2. 避免被低价值噪音带偏
3. 给出了清晰、可执行的修复建议
4. 在需要时输出结构化的逐文件报告和汇总报告
5. 在上下文不足时保持边界意识

## 推荐评分维度

每条 eval 至少按以下维度评估：

1. 关键问题命中
2. 噪音控制
3. 修复建议质量
4. 结论清晰度
5. 格式符合度

对于仓库级或报告型用例，额外评估：

6. 逐文件覆盖完整性
7. 汇总报告完整性

## 评分等级

每个维度使用以下三级判断：

### Pass
- 命中了该用例最关键的问题
- 结论清晰，重点收敛
- 修复建议明确、可执行
- 基本没有被低价值噪音干扰
- 若要求文件级 / 汇总报告，结构完整且满足要求

### Partial
- 命中了一部分关键问题，但遗漏了同等重要的问题
- 或指出了关键问题，但排序不够靠前
- 或修复建议存在但比较泛
- 或夹杂了一些不必要噪音，但未严重影响主结论
- 或报告结构基本有，但不完整

### Fail
- 漏掉了最关键的问题
- 或主要结论被低价值问题占据
- 或没有给出可执行修复建议
- 或在应克制的场景中输出大量低价值意见
- 或在要求逐文件 / 汇总报告时未按要求输出

## 维度细则

### 1. 关键问题命中
关注是否识别出该用例中最值得报告的风险。

- Pass：命中主要风险，且位置正确、优先级合理
- Partial：命中部分主要风险，或命中但排序不够靠前
- Fail：未命中核心问题，或把次要问题当主问题

### 2. 噪音控制
关注是否被格式、命名、风格等低价值问题带偏。

- Pass：主结论聚焦高价值问题，低价值问题极少或没有
- Partial：主结论基本正确，但混入了较多无关小问题
- Fail：主要篇幅都在说低价值问题，或在应克制场景中强行凑意见

### 3. 修复建议质量
关注是否给出低歧义、可执行的修复方向。

- Pass：给出具体改法、边界处理方式或代码级方向
- Partial：有建议，但比较泛泛
- Fail：只有问题描述，没有真正可执行的修复建议

### 4. 结论清晰度
关注输出是否先给结论，再展开关键问题。

- Pass：明确说明是否建议修复后再合并 / 是否未发现值得报告问题
- Partial：能看出倾向，但结论不够收敛或层次混乱
- Fail：没有明确结论，只是堆问题或堆观察项

### 5. 格式符合度
关注输出是否符合该 skill 的预期输出风格。

- Pass：输出结构与用例要求一致，问题包含位置、描述、影响、优先级、修复建议等关键要素
- Partial：大体符合，但缺少若干关键要素
- Fail：结构明显不符合要求，或输出成教程/散文式评论

### 6. 逐文件覆盖完整性
仅用于仓库级或明确要求逐文件报告的用例。

- Pass：每个文件都有判断；有问题文件写清问题，无问题文件也有明确说明
- Partial：覆盖了大部分文件，但有遗漏，或无问题文件没有体现
- Fail：只给全局总结，没有逐文件审核详情

### 7. 汇总报告完整性
仅用于仓库级或明确要求汇总报告的用例。

- Pass：包含总体结论、关键问题、汇总视图、边界说明等核心部分
- Partial：有汇总报告，但缺少关键部分
- Fail：没有汇总报告，或内容无法支持后续分析 skill 表现

## 使用建议

1. 先看“关键问题命中”，这是最重要维度
2. 再看“噪音控制”，避免技能变成低价值意见生成器
3. 再看“修复建议质量”和“结论清晰度”
4. 对仓库级场景，重点检查逐文件覆盖和汇总完整性
5. 不建议把所有维度机械加总成一个 100 分总分；更建议看各维度 pass/partial/fail 分布

## 推荐结果记录格式

可为每条 eval 记录如下结构：

```json
{
  "eval_id": 1,
  "dimensions": {
    "key_findings": "pass",
    "noise_control": "partial",
    "remediation_quality": "pass",
    "conclusion_clarity": "pass",
    "format_compliance": "pass"
  },
  "summary": "命中了关键安全问题，修复建议明确，但混入了少量次要风格意见。"
}
```

对于仓库级用例，可增加：

```json
{
  "file_coverage": "pass",
  "report_completeness": "partial"
}
```
